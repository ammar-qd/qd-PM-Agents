# User Story Generator

This skill helps colleagues turn product input into well-structured user stories in a consistent format.

It is designed for situations where you already have some context — for example from market research, a feature idea, a PRD, a problem statement, stakeholder notes, or raw discovery input — and want to convert that into user stories that are easier to review, refine, and use in planning.

---

## What this skill does

This skill takes open-ended input and converts it into user stories.

It can work with many kinds of input, including:
- market research output
- problem statements
- PRD content
- feature briefs
- discovery notes
- stakeholder requests
- rough ideas
- competitor insights
- product opportunity summaries

Based on the context provided, the skill decides the most suitable structure for the output.

Depending on the scenario, it may generate:
- one user story
- multiple user stories
- an epic with child stories

---

## Who should use this skill

This skill is useful for:
- Product Managers
- GTM colleagues working with product requirements
- stakeholders helping shape feature ideas
- teams translating research or discovery into actionable requirements

---

## When to use this skill

Use this skill when you have product or business context and want to convert it into clear, structured user stories.

Typical examples:
- after using the market research skill and wanting to turn findings into user stories
- after writing a PRD and needing story candidates
- when you have a feature idea and want to break it into stories
- when you have rough stakeholder input and want a cleaner delivery-ready format
- when you want to create Jira-ready draft stories faster

---

## What kind of input it accepts

This skill is intentionally open-ended.

You do **not** need a special format before using it.

You can paste:
- a full PRD
- a market research summary
- a feature concept
- a short paragraph
- bullet points
- meeting notes
- a rough problem statement
- raw pasted text from another tool

The skill will interpret the input and decide how to structure the user story output.

---

## Standard output format

The skill uses this standard format:

```text
USER STORY:
As a [type of user], I want [goal] so that [benefit]

UST DETAILS:
1) [detail]
2) [detail]

ACCEPTANCE CRITERIA:
1) Given that [context], When [action], Then [outcome]
2) Given that [context], When [action], Then [outcome]
